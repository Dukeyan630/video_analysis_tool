# 项目讲解 

## 1.项目背景
从事多年的新媒体运营业务，想要做一个可以通过分析平台、内容类型、发布时间段、互动率等指标来判断哪些内容更值得投入，来帮助创作者优化创作作品。


## 2.技术栈
- Python: 项目主语言
- pandas: 数据导入，清洗和数据分析
- matplotlib: 绘制基础图表
- SQLite: 保存清洗后的数据，并可以用SQL语言查询
- openpyxl: 优化 excel报告格式
- Streamlit: 构建网页可视化图表看板
- Git/github: 版本控制管理和发布展示


## 3.核心流程
用户上传 xlsx或者csv的数据
 ↓
loader 读取数据
 ↓
cleaner 清洗字段、转换类型、生成字段互动率和总互动量
 ↓
analyzer 做平台分析、内容分析、时间段分析
 ↓
plotter/Streamlit 用来展示图表
 ↓
database 录入数据库sqlite，支持sql查询
 ↓
report_generator 导入excel生成数据报表
## 4. 我实现的功能
- 支持xlsx和csv 数据读取
- 支持数据清洗和字段加工
- 自动生成总互动量、互动等级、互动率、播放等级
- 支持平台、内容类型、发布时间范围、互动率筛选
- 支持平台分析、内容类型分析、时间段分析
- 支持 Streamlit 网页看板展示
- 支持导出 Excel 多 Sheet 分析报告
- 支持SQLite数据存储 和SQL语言查询

## 5. 项目难点和解决问题方式
### 1.数据清洗
原始 Excel/CSV 中可能存在空值、字符串数字、列名前后空格等问题。如果不提前处理，后面的求和、平均值和分组分析容易报错。因此在 cleaner.py 中先删除完全空行，统一列名格式，并用 pd.to_numeric() 将点赞、收藏、评论、分享、播放量等字段转换成数字。
### 2.互动率计算中的0值问题
互动率计算方式时，总互动量/播放量，如果播放量为0，可能会遇到除零问题或者异常错误，为了避免使用replace(0,pd.NA)将0替换为空值，避免除以0
### 3.groupby/pivot_table 后表格结构变化
分组字段变成索引，而不是普通列，后期导出excel，或者在plotter，streamlit展示图时不太方便，用reset_index()把索引重新变成普通列
### 4.Streamlit筛选后的数据联动
Streamlit 中用户会动态选择平台、内容类型、日期范围和互动率。因此所有基础概览、平台分析、TOP 排名和图表都应该基于 filtered_df 重新计算，而不是继续使用 cleaned_df。
### 5.Excel报告格式化
pandas 默认导出的 Excel可读性一般，列宽不一定合适，表头滚动后也容易看不见。因此使用 openpyxl 对报告进行格式优化，包括冻结首行、自动调整列宽、设置百分比格式等。
